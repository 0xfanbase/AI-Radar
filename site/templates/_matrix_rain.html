The whole layer is structurally inert, not just visually decorative: - `aria-hidden="true"` on the wrapper removes it from the accessibility tree entirely -- assistive tech never sees it. - `pointer-events: none` (matrix.css) means it can never intercept a click/tap meant for real page content. - `z-index: -1` + `position: fixed` (matrix.css) keeps it strictly behind every in-flow element, including the opaque `background: var(--color-bg)` chrome panels (.masthead, .site-footer, main -- see components.css, T2) that keep it out from behind actual reading text. Every animation property (`--rain-duration`/`--rain-delay` here are inert custom properties, not animations themselves) lives entirely inside matrix.css's own `@media (prefers-reduced-motion: no-preference)` block, with zero fallback animation outside it -- matching board.html's pulse-dot convention exactly. Under reduced motion, columns render as a static, non-animating tiled backdrop. #}
